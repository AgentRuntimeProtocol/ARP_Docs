How-to pages are short, copy/paste recipes for doing **one thing** with ARP.

If you want an end-to-end walkthrough, start with:
- [Quickstart](../getting-started/quickstart.md)
- [Tutorial: Thin Mode (wrap an API)](../getting-started/tutorials/thin-mode.md)

## How-to vs tutorials vs reference

- **How-to Guides**: “do one thing” in a short, copy/pastable sequence of steps.
- **Tutorials**: larger, multi-section walkthroughs.
- **Reference**: precise schemas and APIs.

## Template (what every How-to contains)

1) Goal (“You will …”)
2) When to use this
3) Prerequisites (versions, running services, required tokens/roles)
4) Steps (numbered, copy/pastable)
5) Verify (expected output / checks)
6) Troubleshooting (top 3 failure modes + fixes)
7) Cleanup / Rollback (if relevant)
8) Next steps (Concept + Reference + next How-to)
